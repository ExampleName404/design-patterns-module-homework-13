# Диаграмма вариантов использования: Система управления сетью библиотек

## Ключевые роли
- **Читатель** — конечный пользователь, взаимодействующий с каталогом и бронированиями.
- **Библиотекарь** — расширяет функциональность читателя, отвечает за каталог и выдачу.
- **Администратор** — наследует возможности читателя, дополняет управлением филиалами и учетными записями.

## Основные связи
- `Просмотр и поиск книг` <<include>> `Регистрация пользователя` — гость сначала регистрируется.
- `Бронирование книги` <<extend>> `Просмотр и поиск книг` — бронирование доступно после поиска.
- `Выдача/возврат книг` <<include>> `Бронирование книги` — использование броней при выдаче.
- `Управление книгами` <<include>> `Выдача/возврат книг` — работа с каталогом дополняет выдачу.
- `Просмотр аналитики` <<extend>> `Управление филиалами` — аналитика доступна администраторам в рамках управления.

## Mermaid диаграмма
```mermaid
%%{init:{'theme':'neutral'}}%%
usecaseDiagram
actor Reader as "Читатель"
actor Librarian as "Библиотекарь"
actor Admin as "Администратор"
Reader <|-- Librarian
Reader <|-- Admin
usecase UC1 as "Регистрация пользователя"
usecase UC2 as "Просмотр и поиск книг"
usecase UC3 as "Бронирование книги"
usecase UC4 as "Управление книгами"
usecase UC5 as "Выдача/возврат книг"
usecase UC6 as "Управление филиалами"
usecase UC7 as "Просмотр аналитики"
Reader --> UC1
Reader --> UC2
Reader --> UC3
Librarian --> UC4
Librarian --> UC5
Admin --> UC6
Admin --> UC7
UC2 ..> UC1 : <<include>>
UC3 ..> UC2 : <<extend>>
UC5 ..> UC3 : <<include>>
UC4 ..> UC5 : <<include>>
UC7 ..> UC6 : <<extend>>
```

## Текстовое описание
1. **Читатель** регистрируется, просматривает каталог, ищет книги, оформляет и отменяет бронирования, отслеживает историю.
2. **Библиотекарь** выполняет операции читателя и дополнительно ведёт каталог, фиксирует выдачу/возврат, следит за активными бронями.
3. **Администратор** наследует возможности читателя, управляет филиалами и учетными записями сотрудников, просматривает аналитические отчёты.
